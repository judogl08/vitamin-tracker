Vitamin Tracker v5 — исправление обновления

Причина проблемы в v4:
service worker пытался кэшировать удалённые файлы icon-192.png и icon-512.png.
Из-за ошибки cache.addAll новый service worker мог не установиться, поэтому Safari
продолжал использовать старую страницу.

Исправлено:
- корректный список файлов кэша;
- новый кэш vitamins-v5;
- skipWaiting() и clients.claim();
- network-first для страницы и файлов;
- отключено использование HTTP-кэша при проверке обновлений;
- новые имена иконок v5;
- к CSS, JS и manifest добавлен параметр ?v=5;
- внизу приложения отображается «Версия 5.0».

Для GitHub:
загрузите все файлы из этой папки в корень репозитория с заменой существующих файлов.
